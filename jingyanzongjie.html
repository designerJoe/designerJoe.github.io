因为面试官在看纯视觉稿的简历时，完全不知道你在设计的过程中是如何思考的，所以这是非常重要的一步，能不能脱颖而出就看这里。

04.改版理由

你的设计稿很可能已经是好几个迭代之后的产物了。这时候你可以在之前的基础上，写上项目的改版理由。这样不仅可以让面试官觉得你在项目中有去思考问题，还能帮助自己更好的去梳理和复习项目流程，不会在面试的时候一时想不起关键内容。

以上这些截图是从部分简历以及朋友重设计的案例中截取出来的，为了保障隐私我隐去了一些内容。类似的问题其实还有很多，我就不一一举例了。最后再说说面试官更想在你的简历中看到哪些内容。

05.工作经历

如果可以，尽可能详细的描述你的工作职责，这样可以让面试官对你的过去有一个初步的印象。

比如：2006年8月 - 2008年9月 某某公司 UI 设计师，负责公司核心产品的视觉设计，包括界面、图标、卡通形象的设计。2008年10月 - 2010年12月 某某公司 UI 设计师，负责公司多款产品的视觉及交互设计，是三个 UI 设计师的负责人，分配设计师工作时间及管理项目工时，定期做团队分享，与其他同事一起学习进步。等等。

当中还可以有很多更为详细的内容，尽可能的展示出自己的优势，让人感觉到这是一位发展不错的设计师，并且自律性强。表现出一直不断的在提升自己，会让人对你更有兴趣。

06.个人经历

这个与工作经历不同，可以讲述一些自己平时的兴趣爱好，或者业余会做的事。比如我，除了工作，大部分时间都在写文章、看书，所以积累了一定的知识体系，这个优势可以让你在面试中加分。

因为一个团队会更注重你的学习欲望及能力，所以有时候做一些其他功课，会让你更具优势，把玩游戏的时间省下来，可以做的更多。

07.项目案例解析

从众多项目中，拿出一个点来解析你的工作 Flow，你是如何很好的完成这个项目的。从你的工作流程、设计思路、论证方法以及展示解决商业问题的能力来打动面试官，把面试官当做你的用户。只有把以上七点做到位，面试官才有可能认真仔细的看你的简历，而不是随意浏览一遍之后就被刷走了。

设计师不是仅仅在需要设计的地方去做设计，而是把不需要设计的东西都设计好了，才能被称为一个合格的设计师。
